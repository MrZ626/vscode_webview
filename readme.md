# VS Code Webview

> **声明：该项目是我（MrZ_26）的第一个实验性 Vibe Coding 项目，所有技术细节几乎均由Claude Opus 4.6完成**  

## 项目动机

我想在安卓设备上使用 VS Code 作为文本（代码）编辑器，外出可以只带键盘不用带笔记本电脑

用浏览器打开的话可能会有快捷键冲突，那我想单独打包成一个app就能避免

## 项目功能

webview打开vscode.dev链接，没了

然后由于不是浏览器环境，光一个webview并无本地文件读写功能，那这菜过头了肯定不行，于是让AI把打开文件（夹）请求做了桥接，实现了基本的文件读写功能

注：文件和文件夹的增删改还是有点工作量的，AI也犯错修改了好几轮

## 猜你想问

- 不能登录，那怎么导入设置和快捷键？
    1. 在电脑vsc命令面板找到`Open User Settings (JSON)`打开`settings.json`，把这个文件以及和它同目录的`keybindings.json`拷到安卓设备上
    2. 安卓设备打开本应用，从命令面板打开`settings.json`（它应该是在应用的内部目录里，确实有目录），把拷过来的文件内容全选复制粘贴保存
    3. 保持在打开本机`settings.json`的状态，虽然无法打开它所在的文件夹，但注意到标签页底下显示了它的路径`User > settings.json`，点击那个User会出现下拉框里面有`keybindings.json`（其实一开始没有，你得手动随便改一个快捷键然后就有了），点它，复制粘贴保存完事
- 各种联网功能都用不了，怎么打开应用还要网？
    - 不懂，但我觉得在浏览器里顺手按个Ctrl+W直接给整个应用关了更烂，我宁愿用这个需要联网才能打开的离线app（
    - 这apk总共就30KB，不觉得这很酷吗，很符合我对安卓连个像样的代码编辑器都没有的生态的想象，科技并带着趣味
- 打不开图片？
    - 调试结论是vsc打开图片压根就没走文本文件读写流程
    - 算了，反正我们是来打字的，真要看图就去系统文件管理器里看吧
